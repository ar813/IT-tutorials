1. ORDER BY: ordering data like ascending order(ASC) or descending(DESC)

	SELECT * FROM table_name ORDER BY
	col_name ASC

	col_name DESC

2. DISTINCT: Only filters unique data

- It removes duplicate rows from the result.
- For example, if you have multiple students and many of them are from the same city like Karachi, Lahore, or Multan, and you want to list the cities only once, you can use DISTINCT.

	SELECT DISTINCT col_name FROM
	table_name

	SELECT DISTINCT age FROM students

3. ORDER BY and DISTINCT are together

	SELECT DISTINCT age FROM students ORDER BY age ASC

INNER JOIN: INNER JOIN is used to return only the matching rows from both tables based on a common column.

‚úÖ Example: Suppose you have two tables:

1. Students

student_id	name
1	        Arsalan
2	        Ahmed
3	        Fatima

2. Marks

student_id      subject	        marks
1	        Math		95
2	        Math		85
4	        Math		90

üìå SQL Query:

SELECT Students.name, Marks.subject, Marks.marks 
FROM Students
INNER JOIN Marks ON Students.student_id = Marks.student_id;

üì§ Output:

name		subject		marks
Arsalan		Math		95
Ahmed		Math		85

- Syntax

	SELECT columns
	FROM table1
	INNER JOIN table2 ON condition


************* üñ•Ô∏è Lesson Twenty-one *************


LEFT JOIN: returns all records from the left table (table1), and matched records from the right table (table2) 
	
	SELECT column_name(s)
	FROM table1
	LEFT JOIN table2 
	ON 
	table1.column_name = table2.column_name

RIGHT JOIN: returns all records from the right table (table2), and matched records from the left table (table1) 

	SELECT column_name(s)
	FROM table1
	LEFT JOIN table2 
	ON 
	table1.column_name = table2.column_name


************* üñ•Ô∏è Lesson Twenty-two *************

CROSS JOIN: CROSS JOIN returns the Cartesian product of two tables ‚Äî it combines every row of the first table with every row of the second table.

	SELECT column_name(s)
	FROM table1
	CROSS JOIN table2 
	ON 
	table1.column_name = table2.column_name
